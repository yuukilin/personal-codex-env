# Advanced Research Report — 進階決策型研究

## 定位

把使用者已經初步認識的標的研究到可以做決策。報告不只列出 thesis、新聞、利多與風險；必須把最重要的事件追到「發生多嚴重、透過什麼機制影響、何時反映、可能影響多大、接下來看什麼」。

輸出採 thesis-first，先告訴使用者結論；研究過程不可 thesis-locked。先建立暫定假說、替代解釋與反方假說，再用證據淘汰或修正，最後才確定核心 thesis。

若使用者只是第一次建立認識、做公司或產業入門，改用 `research-report`。

## 必讀參考檔

每次執行都完整讀取：

1. `references/deep-analysis-core.md`：共用深挖方法、證據規則與完成門檻。
2. 依標的只選一個模板：
   - 上市公司：`references/decision-report-template.md`
   - 商品、農產品、能源、金屬：`references/commodity-deep-dive-template.md`

不要同時載入兩個資產模板，避免不相關規則占用研究空間。

## 執行原則

- 先確認今天日期；涉及股價、財報、新聞、政策、產品規格、供需、天氣、產量、庫存、目標價與估值時重新查證。
- 全文使用繁體中文。假設讀者對標的沒有完整背景；技術、理工、金融與農業機制先用高中生能理解的方式說明。
- 優先使用 Obsidian MCP 讀寫 vault。若工具未露出，先用 `tool_search` 搜尋 `obsidian mcp tools` 或 `mcp__obsidian_mcp_tools`，再呼叫 `get_server_info` 驗證；只有載入或驗證失敗時，才明確說明限制並使用本機 vault fallback。
- 高風險或時間敏感的核心判斷至少交叉查 3 個可靠來源；公司/官方原始資料或財報狗可見的同一筆股價、財報、估值、財務指標與美股法說逐字稿不必為湊數重複查 3 次，但重要預估、因果推論與爭議性說法仍要尋找獨立證據。
- 表格只用於壓縮比較結果，不可取代因果敘事。主文原則上不超過 3 張表；每張重要表格後用段落解釋「真正代表什麼」。
- 把研究篇幅集中在最重要的 2–3 個驅動因素，不要列十個平均重要、各寫一兩句的催化劑。
- 還原漲跌原因時先鎖定最新價格時間與市場收盤時間；收盤後才發布的新聞只能列為「尚未交易的新催化劑」，不可倒過來解釋先前漲跌。個股與商品都適用。
- 個股漲跌先做歸因分流：公司新資訊、產業/同業共振、大盤/利率/地緣風險、籌碼與財報前卡位。先查同日公司 IR/SEC 是否真的有新公告，再看相對同業與大盤的超額報酬；沒有公司級新資訊時，明寫「這次主要是族群或市場重定價」，不得拿數週前舊消息硬套成當日催化。
- 找不到關鍵估算的原始來源、方法或基準時，寫「我搜到的資料不足以確認」，不得把它放進核心 thesis 或 base case。
- 精確價格、停損點、目標價或爆發日期必須附推導；使用者沒有要求交易計畫時，不要讓操作價位占據研究主體。
- Markdown 表格內不用 `[[頁面|顯示文字]]` alias wikilink；改用純文字或不含 alias 的 wikilink。

## 工作流程

### 1. 判斷標的與真正問題

確認標的是公司或商品，抽取使用者最想回答的 1–2 個問題。若使用者沒有提供部位，預設為重新評估是否值得建立或持有部位，不停下來詢問。

常見核心問題：

- 最近到底為什麼漲跌？最重要驅動因素是什麼？
- 這個事件有多嚴重，會持續多久，影響可能有多大？
- 撇除短期事件，長期供需或公司競爭力正在變好還是變壞？
- thesis 是否已被價格反映？最可能何時由財報、產量或市場數據驗證？

### 2. 建立待驗證假說，不先下結論

研究開始時先寫工作底稿，不直接放進報告：

- 暫定主假說：目前最可能的解釋。
- 替代解釋：同一現象還可能由什麼造成。
- 反方假說：若市場正在誤判，最可能錯在哪裡。
- 驗證資料：哪些原始資料能區分三者。
- 翻案條件：看到什麼就必須改判斷。

### 3. Vault 圖譜檢索

先讀 `_system/rules/obsidian-rules.md`，再依圖譜順序搜尋：

1. 個股從 `3 MOC/Tickers/`；商品或主題從 `1 Cards/Theme-*`、tracker 與產業 MOC 起手。
2. 讀近期 `3 Analysis/Signals/`、`3 Analysis/Daily/`、`_mini-summaries/` 與趨勢演變。
3. 沿 outgoing links、backlinks、frontmatter 找到高相關來源。
4. 主角公司、最新法說/財報、核心商品官方供需與本輪前三大催化劑來源完整讀；同業與旁支先篩選再深讀。
5. 對照上次 thesis，記錄哪些變好、變差、尚未改變，以及舊報告哪些問題仍未回答。
6. 補查最近 90 天同主題但尚未完整吸收到 MOC、Theme 或 tracker 的新來源，避免只沿舊圖譜而漏掉最新轉折。

若公司、品牌、子公司或事業部的證據藏在母公司法說/簡報/Q&A，執行來源升級，完整讀相關上下文，不只摘命中句。

超過 6 個月的具體股價、財測、供需、產量與估值只能作歷史脈絡；若要當現在判斷，必須重新查證。

### 4. 外部資料與來源升級

共通優先順序：公司或官方原始資料、交易所/監管資料、專業產業或田間資料、可信券商研究、可靠新聞、其他推論。

個股：

- 台美股若財報狗有涵蓋，財報狗是股價、財報、財務指標、估值與美股法說逐字稿的核心來源。用已登入 Chrome 開受控分頁，等待載入、確認畫面，再讀文字或視覺表格；預期應有資料但因登入或方案權限不足時停止並回報，不用其他來源假裝補完。若財報狗本來就未涵蓋該市場或標的，改用公司 IR、交易所與監管原始資料，不把「未涵蓋」誤判成權限故障。
- 同時讀最新公司 IR、財報、法說完整上下文、監管申報、競爭對手與上下游資料。
- 產業報告只支持類別時，不可升級為公司特定證據；必須找到產品、客戶、認證、訂單或出貨連結。
- 原文出現 support、interest、explore、forecast、customer demand、commitment、order、backlog、shipment 或 revenue 時，保留原詞並分級；不得把客戶預測或 demand 自動改寫為訂單。若管理層同時揭露可供應量或公司 outlook，base case 以較低的 supply-backed revenue 為準。
- 最新財務不只看成長率，還要檢查絕對基期、關係人/一次性大單、客戶集中、ACV/RPO/backlog、現金流與認列時點；成長率很高但未揭露絕對金額時，只能確認方向，不能自行推重大性。
- 若公司獲利直接曝險商品或原料週期，強制建立「區域基準報價 → 公司實現價格 × 銷量 → 可用產能與利用率 → 產品組合 → 單位原料/能源成本、維修與停機 → EBITDA/自由現金流」橋接。分清總產能與可用產能、現貨與合約、全球與區域、報價月份與財報認列落差；基準商品下跌但股價上漲時，必須另查基差、資本回饋、同產品同業與公司特定事件。
- 週期股估值至少同時呈現最新十二月與正常化/中景氣獲利，排除一次性收益、股數變化與不可自由支配的合資現金流；不得只因 headline P/E 或當期 EV/EBITDA 低就判斷便宜。

商品：

- 優先讀官方供需/庫存、產地政府或產業機構、氣象與田間調查、港口/出口/研磨/壓榨資料。
- 一般災情報導只能證明災情；要推到產量，必須再確認產區重疊、作物階段、田間損害與持續時間。
- 把目前作物、下一作物年度與長期結構分開，不可混用到港量、預估產量與庫存年份。
- 對天氣、停產、罷工與物流事件，分開回答「事件還會持續多久」與「已造成的影響會留多久」；無可靠短期預報時明說不知道，不用季節預報假裝能精確到未來幾週。

### 5. 執行共用深挖

依 `references/deep-analysis-core.md` 對前三大驅動因素逐一完成：事件嚴重度、位置/業務範圍、因果機制、量化重大性、時間軸、替代解釋、確認指標與反證。

每個重要事實寫完都追問：

> 然後呢？會持續多久？何時反映？影響可能多大？接下來看什麼？

若四題有兩題以上答不出來，該項只能列為初步訊號，不能視為完成研究。

### 6. 套用資產模板並形成最終 thesis

完成證據蒐集後才決定：

- 最重要 thesis 與狀態：變好、變壞、持平或資料不足。
- 最強替代解釋與反方 thesis。
- 市場已反映多少，以及判斷依據。
- 最可能驗證窗口與逐步里程碑。
- 行動判斷、信心等級、代價與翻案條件。

公司要把事件接到產品/客戶、商業化階段、可供應量、部門營收、公司獲利與估值。新舊業務毛利率、資本強度或認列模式不同時，必須分開建模，不得沿用舊業務 margin。商品要把事件接到產區/作物階段、產量、全球平衡表、庫存與價格。

### 7. 完成驗收

逐項通過 `references/deep-analysis-core.md` 的完成門檻。關鍵資料不足時，不硬湊完整結論；把行動改成「等待資料」，明確列出缺口與下一個確認點。

### 8. 存入 Obsidian

完成後直接存入 `2 Sources/Research/`：

- 檔名：`YYYY-MM-DD-進階研究-[標的名稱].md`
- frontmatter 至少包含 `date`、`tags`、`source`、`status: draft`，再依模板加入研究欄位。
- 使用 `[[wikilink]]` 連結既有 Theme、Ticker、Signal、tracker 與來源筆記。

若 Obsidian MCP 不可用，存到本回合可寫的輸出資料夾並說明限制。

## 完成回報

回覆使用者時簡短列出：

1. 筆記位置。
2. 核心 thesis、狀態與是否已反映。
3. 最重要的 2–3 個深挖結論。
4. 行動判斷、信心與最大反證。
5. 仍不知道什麼，以及下一個確認點。

## 禁用事項

- 禁止把「具備技術/產能」直接寫成「已取得訂單/即將貢獻營收」。
- 禁止把生態支持、探索合作、客戶興趣、forecast 或 customer demand 改寫成 firm order、backlog 或可認列營收。
- 禁止用數週前舊公司消息解釋當日股價，卻沒有先核對同日公告、同業與大盤共振。
- 禁止只看年增率而不查絕對基期、關係人/一次性收入與訂單能見度；禁止把新業務套用舊業務歷史毛利率。
- 禁止用單一全球商品報價直接代表公司的區域實現價，或用「可用產能利用率」假裝全部設計產能都在運轉；禁止用週期高點的 EPS/EBITDA 直接判斷低估。
- 禁止把產業類別緊俏直接寫成特定公司受惠。
- 禁止把一般災情直接寫成農作物減產。
- 禁止引用「部分分析師預估」卻不追查具名來源、日期、基準與方法。
- 禁止只有表格、bullet point 與一兩句結論，沒有完整因果敘事。
- 禁止只列利多，不處理替代解釋、相反資料與未知數。
- 禁止用過期數據或無推導的精確目標價、價格區間與爆發期。
- 禁止需要財報狗卻看不到資料時硬寫個股報告。
