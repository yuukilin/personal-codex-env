## 1. 研究目的與資料源

商品報告優先回答兩題：

1. 最近到底為什麼漲跌？前三大驅動因素與價格傳導是什麼？
2. 撇除短期事件，未來一個作物/供應年度與 3–5 年整體供需結構如何？

資料優先順序：

1. 官方供需、庫存、產量、出口、氣象、交易所與監管資料。
2. 產地政府、產業協會、田間/礦區/航運調查與具名專業研究。
3. 可靠的加工、研磨、壓榨、進出口、港口與企業採購資料。
4. 可信新聞與訪談。
5. 未具名市場說法、同類商品推論與社群觀點。

一般災情、天氣或政治新聞只能證明事件存在；要推到供給與價格，仍要證明產區重疊、生產階段、實際損害、持續時間與全球重大性。

## 3. 正文結構

### 一、先回答：最近在漲跌什麼，長期又是什麼

用 3–6 個完整段落回答：

- 最近這段價格變化最重要的第一、第二、第三驅動因素。
- 哪個是基本面主因，哪個是導火線，哪個只是倉位或宏觀助燃劑。
- 市場交易的是眼前現貨、下一年度風險，還是長期結構。
- 撇除短期事件，供需安全邊際正在改善或惡化。
- thesis 是否已被價格反映，現在的行動與最大反證。

不要用一張 thesis 表格取代回答。

### 二、最低必要背景

假設使用者沒有完整商品背景，先講懂：

- 全球主要產國/供應地、加工地與需求地，各占比大約多少。
- 作物年度、主收穫季/小收穫季，或礦山/能源生產與庫存週期。
- 目前市場看的期貨合約月份與它代表的供給期間。
- 產量、到港、出口、庫存、加工量與消費量分別代表什麼。

這一節只補理解本輪行情所需知識，不寫百科全書。

### 三、價格時間線與驅動因素排序

按日、週、月三個時間窗還原：

- 事件發生日期與價格反應。
- 價格、成交量、未平倉量與可得的現貨/期貨曲線變化。
- 同期美元、能源、運費或相關商品是否共同影響。
- 基本面新聞是在漲前、漲中或漲後出現，避免把事後新聞誤當成起因。
- 鎖定最新價格的日期、時間、時區與合約；晚於該價格截止點才發布的資料，另列為「尚未交易的新催化劑」，不可納入已發生漲跌的歸因。

排序時使用「核心驅動／重要助力／放大器／背景因素」，不要假裝能精確分配百分比，除非有可驗證模型。

### 四、前三大催化劑深挖

每個催化劑完成 `deep-analysis-core.md` 的十問，尤其追查：

- 事件到底發生在哪些主要生產區，而不只是首都或城市。
- 相對季節正常值、歷史分位或原預估有多異常。
- 發生時處於開花、幼果、成熟、收割、加工、運輸或庫存週期哪一段。
- 影響是短期物流、品質折損、可逆延誤，還是永久產量損失。
- 天氣、罷工、政策或停產預計持續多久；事件結束後是否仍有病害、樹勢、良率或維修延遲。
- 將事件本身的持續時間，與不可逆、路徑依賴、可逆三類後果分開。沒有可信短期預報時明說未知，不能用月度或季節預報回答未來數週。
- 田間、pod count、礦區、港口、出口或企業採購資料是否已確認。
- 產量估計由誰、何時、用什麼方法算出，其他機構的區間如何。
- 田間調查的樣本數、覆蓋產區與代表性如何；自然落果、正常停工或基準損耗是多少，避免把異常比例直接當成全國減產比例。

前三大催化劑要用敘事解釋「之後會怎樣」，不可只列事件名稱與方向。

### 五、近端供需：未來 1–3 個月

分開處理：

- 目前可交割或可出口供給。
- 港口、運輸、品質與加工瓶頸。
- 交易所、產地與下游庫存。
- 需求端採購、研磨/壓榨、替代與高價需求破壞。
- 投機倉位與空頭回補是否只是放大器。

遇到「到港量強但下一季看壞」等矛盾，先檢查是否在談不同作物年度或流量/存量，再清楚解釋兩者如何同時成立。

### 六、中長期供需：一個年度與 3–5 年

至少回答：

- 過去 5–10 年全球產量、需求、盈餘/赤字與庫消比如何變化。
- 本年度到下一年度，哪些國家增加或減少，對全球平衡各貢獻多少。
- 主要生產區的單產、樹齡/礦山品位、病害、農民/礦商投資、政策與成本。
- 新產區、新礦、替代供應需要多久才能補上。
- 高價造成的需求破壞、配方替代、回收或效率提升是暫時還是結構性。
- 全球供給集中度與庫存安全邊際是否使市場更容易受到下一次衝擊。

不要只列兩個年度的全球總數；要解釋變動由哪個國家與機制造成。

### 七、全球平衡橋與關鍵估算

建立平衡橋：

| 項目 | 當前基準 | Bear | Base | Bull | 來源與假設 |
|---|---:|---:|---:|---:|---|
| 主要國家供給變化 | | | | | |
| 其他產區抵銷 | | | | | |
| 全球需求／加工變化 | | | | | |
| 全球盈餘／赤字 | | | | | |
| 期末庫存／庫消比 | | | | | |

表後用完整段落說明：哪個輸入最影響結論、其他產區能補多少、需求破壞能抵銷多少，以及目前公開資料能否支持精確估計。

所有核心產量預估交代具名來源、日期、基準、方法與範圍。無法溯源的市場數字不得進 base case。

主產季、小產季與完整作物年度必須分欄或分段；不同年度的盈餘、庫存或產量只能做量級比較，不能直接宣稱下一年度已轉成赤字。若缺其他產區與需求假設，就只給方向式情境，不填完整全球平衡表。

### 八、倉位、市場結構與是否已反映

檢查：

- 現貨與期貨價格、近遠月曲線、基差或升貼水。
- 交易所、商業庫存與產地庫存的差異。
- COT、未平倉量、成交量與多空變化；區分新多、空頭回補與多單撤退。
- 事件前後不同合約與相關市場的反應。
- 價格目前大概反映了哪個供需情境。

只有淨多或淨空一個數字不足以判斷擁擠；只有價格上漲也不能證明實體短缺。

|---|---|---|---|---|
| Bear | | | | |
| Base | | | | |

價格區間必須連到平衡表、庫存、歷史類比或市場結構。無法合理推導時，只給方向與條件，不給假精準價格。

使用者未要求交易計畫時，主文不要提供細碎進出價。若使用者明確問操作，另加附錄說明合約、波動、部位、停損推導與最大損失。

### 十、替代解釋與反方 thesis

至少處理：

- 最近漲跌是否主要來自倉位、美元或技術面，而不是實體供需。
- 市場是否把未確認的下一年度風險提早交易過頭。
- 其他產區增產與需求破壞能否抵銷主產區損失。
- 目前資料是否只是短期物流延誤，而非永久產量損失。

列出至少 3 個具體翻案條件。

### 十一、接下來會怎樣：驗證日曆

依即時、1–3 個月、一個供應年度列出：

- 下一次天氣/產區/礦區資料。
- 到港、出口、庫存、加工或產量更新。
- 官方供需報告與方法透明的私人預估。
- 需求端企業財報、採購與替代訊號。
- 每個資料的判斷門檻，以及結果出現後如何升降 thesis。

### 十二、資料來源、Vault 覆蓋與未知數

列出實際使用來源、日期、用途與證據等級。說明讀過哪些 Theme、tracker、Daily、Signal 與歷史報告，以及：

- 舊 thesis 和本次差異。
- 來源互相衝突之處。
- 目前仍不知道什麼。
- 下一輪最需要補的田間、官方或需求資料。

## 4. 商品專用深挖規則

- 天氣：分開「雨量異常」「城市災情」「產區受災」「田間損害」「最終減產」。每一步都要證據。
- 農產品：交代作物階段、受損是否可逆、病害潛伏與下一季樹勢；不能只寫太乾或太濕。
- 金屬/能源：交代礦山/油田、品位/產量、檢修、制裁、運輸、庫存位置與替代供應。
- 政策：分清公告、執行、合規緩衝期、實際貿易流改變與庫存提前囤積。
- 需求：區分終端消費、加工量、補庫、去庫與高價造成的替代。

## 5. 決策規則

- 做多／加碼：供需惡化有產區或實體數據確認，庫存安全邊際薄，價格尚未完全反映 base case。
- 續抱：供需 thesis 未破壞，但價格已反映較多或等待下一個產量/庫存確認。
- 減碼：好消息多已反映、倉位擁擠，或實體數據沒有跟上價格。
- 退出：核心供給損失被推翻、其他產區充分抵銷，或需求破壞使全球平衡顯著改善。
- 觀望／等待資料：只有新聞或模型風險，尚無產區、田間、出口、庫存或加工確認。

## 6. 品質檢查

除共用完成門檻外，再確認：

- 已回答「最近到底在漲跌什麼」與「長期供需怎麼看」。
- 一般洪災、乾旱或政治事件沒有直接被寫成減產。
- 核心產量預估可追到具名來源與方法。
- 價格截止點晚於所有已歸因催化劑；收盤後新資料已明確標示尚未交易。
- 主產季/全年、實際/預測、流量/庫存與公司代理/全球需求沒有混口徑。
- 天氣事件持續時間與損害持續時間已分開；短期預報不足時已明講未知。
- 已區分目前作物、下一作物年度與長期產能。
- 已將主要國家變化接回全球平衡與庫存安全邊際。
- 倉位、美元與技術面被視為驅動或放大器，而非未經驗證的唯一原因。
- 精確價格區間有平衡表或歷史依據；否則改成條件式方向。
