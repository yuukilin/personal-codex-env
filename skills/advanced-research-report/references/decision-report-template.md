# 進階決策型研究報告模板

## 研究依據

這個模板吸收三種報告思維：

- 監管文件思維：公司 10-K/年報與 10-Q/季報提供 business、risk factors、MD&A、financial statements，是事實底稿，不是二手評論。
- 估值思維：好的 thesis 必須能落到營收、利潤率、現金流、資本需求與估值假設；故事和數字對不上時，要先懷疑 thesis。
- 券商實務思維：高品質報告通常先給結論與 rating/target/upside，再解釋財測修正、actual vs forecast、產業週期、受惠者排序、催化劑與風險。
- 使用者資料源偏好：財報狗是股價、財報、財務指標、估值與美股法說逐字稿的核心可信來源。財報狗能看到的資料不需要為同一數字交叉驗證 3 個來源。財報狗採已登入 Chrome 優先：用使用者既有已登入 Chrome 狀態開受控分頁讀取，讀完關閉分頁；不要把未登入公開頁或一般 web fetch 當主流程。財報狗動態頁要先等載入、看畫面、再讀文字；DOM/table count 只能輔助，不能用來判定資料不存在。美股法說會逐字稿優先從 `https://statementdog.com/analysis/{TICKER}/e-report` 找 `/earnings_calls/{id}`。

常用公開來源：
- Investor.gov Form 10-K 說明：https://www.investor.gov/introduction-investing/investing-basics/glossary/form-10-k
- SEC EDGAR 公司申報搜尋：https://www.sec.gov/edgar/search/
- Aswath Damodaran 估值資料與課程：https://pages.stern.nyu.edu/~adamodar/New_Home_Page/home.htm
- Berkshire Hathaway shareholder letters：https://www.berkshirehathaway.com/letters/letters.html

## 報告標題

```markdown
# [公司名稱]([股票代號]) 進階研究報告

> 報告日期：YYYY-MM-DD
> 資料截止：YYYY-MM-DD HH:MM（時區）
> 最新價格：數字（來源，時間）
> 核心 thesis：一句話
> Thesis 狀態：變好/變壞/持平/資料不足
> 決策問題：買入/續抱/加碼/減碼/賣出/觀望
> 報告類型：進階決策型研究
```

## 正文結構

### 一、Thesis 摘要

這是整份報告最重要的一節。先回答 thesis，再談買賣。

| 項目 | 判斷 |
|------|------|
| 最重要 thesis | 一句話說清楚 |
| Thesis 現況 | 變好/變壞/持平/資料不足 |
| 主要證據 | 3 點以內 |
| 是否已反映 | 未充分反映/大致反映/過度反映/不確定 |
| 爆發期 | 例如 2026H2、2027Q1、未明 |
| 最大反證 | 1-3 個數據或事件 |

寫法要求：
- 不要把 thesis 寫成公司介紹，例如「公司是某某龍頭」不夠。
- thesis 要像「如果 X 發生，公司的 Y 財務指標會在 Z 時間開始加速，因此市場可能重新定價」。
- 爆發期不是亂猜，要連到可觀察事件，例如新產品放量、客戶導入、法說 guidance、月營收轉折、毛利率改善、產能開出。

### 二、決策摘要

用一頁內回答：

| 項目 | 判斷 |
|------|------|
| 行動建議 | 買入/續抱/加碼/減碼/賣出/觀望/等待資料 |
| 信心等級 | 高/中/低 |
| 投資期間 | 例如 3-6 個月、12 個月、2-3 年 |
| 核心理由 | 3 句內 |
| 最大風險 | 1-2 句 |
| 需要翻案的證據 | 1-3 個條件 |

摘要必須明確，但不要裝成保證。若資料不足，行動建議改成「等待資料」，並說缺什麼。

### 三、這次和上次認知相比改變了什麼

整理「變好」「變差」「沒有改變」：

| 面向 | 最新變化 | 對 thesis 的影響 | 來源 |
|------|----------|------------------|------|
| 需求 | | | |
| 供給/產能 | | | |
| 財務 | | | |
| 競爭 | | | |
| 估值/股價 | | | |

這一節要緊扣第一節 thesis。使用者已經知道公司基本介紹，所以重點是新資訊與舊 thesis 的差異。

### 四、公司目前狀況

說明公司最新季度與管理層說法：

- 營收、毛利率、營益率、EPS、自由現金流與負債變化。
- segment 或產品線誰在成長、誰在拖累。
- guidance 或管理層展望是上修、下修還是維持。
- 最新產品、客戶、訂單、產能、價格或成本變化。

每個數字標注日期與來源。不要用舊財報數字代表最新狀況。若財報狗可見，股價、財報、估值、財務指標與美股法說逐字稿優先引用財報狗。讀財報狗時先用 Browser/Chrome 連到使用者既有已登入 Chrome 狀態，開受控分頁，等資料載入約 5 秒，看畫面是否有圖表或「詳細數據」，再讀文字層；讀完關閉分頁。不要把未登入公開頁或一般 web fetch 當主流程。若文字抽不到但畫面有表格，改用表格區截圖/視覺讀表，來源標成「財報狗畫面表格」。若美股需要法說會逐字稿，到 `https://statementdog.com/analysis/{TICKER}/e-report` 打開「法說會逐字稿」，優先引用英文原文，中文摘要與 AI 翻譯只作輔助。

### 五、產業上中下游與競爭格局

回答公司所在位置和外部環境：

| 環節 | 目前狀況 | 對公司影響 | 主要公司/資料來源 |
|------|----------|------------|-------------------|
| 上游 | 原料、設備、供應瓶頸、成本 | | |
| 中游 | 製造、產能、良率、價格 | | |
| 下游 | 客戶需求、庫存、預算、採購週期 | | |

競爭對手比較：

| 指標 | 本公司 | 競爭對手 A | 競爭對手 B | 判讀 |
|------|--------|-------------|-------------|------|
| 成長率 | | | | |
| 毛利率/營益率 | | | | |
| 市占/客戶 | | | | |
| 技術/產品 | | | | |
| 估值 | | | | |

### 六、財務品質與估值

最低限度要看：

- 成長：營收 YoY/QoQ、segment 成長、backlog 或訂單。
- 獲利：毛利率、營益率、EPS、營業槓桿。
- 現金流：營業現金流、自由現金流、資本支出、庫存與應收款。
- 財務安全：現金、負債、利息保障、稀釋風險。
- 報酬率：ROIC/ROE 是否改善。
- 估值：PE、EV/EBITDA、P/S、P/B、FCF yield 或 DCF，依產業選擇，不要硬套單一倍數。

估值表：

| 情境 | 主要假設 | 合理估值/目標區間 | 相對目前價格 | 成立條件 |
|------|----------|-------------------|--------------|----------|
| Bear | | | | |
| Base | | | | |
| Bull | | | | |

若無法合理估值，要明確說資料不足，改用「等待哪個數據」取代假估值。

### 七、投資論點與反證

把論點寫成可被驗證的句子：

| 論點 | 支持證據 | 反證 | 目前判斷 |
|------|----------|------|----------|
| 論點 1 | | | |
| 論點 2 | | | |
| 論點 3 | | | |

至少列出 3 個反證。反證不是形式化風險，而是「如果發生，就應該承認 thesis 錯了」的條件。

### 八、催化劑與追蹤指標

| 時間 | 事件/數據 | 為什麼重要 | 對買賣判斷的影響 |
|------|-----------|------------|------------------|
| 1 個月內 | | | |
| 1-3 個月 | | | |
| 3-12 個月 | | | |

追蹤指標要具體，例如月營收、book-to-bill、毛利率、庫存天數、雲端 capex、客戶拉貨、產品認證、法說 guidance，而不是「持續觀察產業」。

### 九、買賣決策

依據情境給結論：

- 買入：上行空間明顯大於下行，關鍵假設已有足夠證據，估值尚未完全反映。
- 加碼：原 thesis 變強，價格或估值仍合理，且新證據提升勝率。
- 續抱：thesis 尚未破壞，但新買的安全邊際不夠。
- 減碼：好消息大多已反映，或風險報酬轉差，但 thesis 尚未完全破壞。
- 賣出：核心 thesis 被數據推翻，或估值已嚴重超過合理情境。
- 觀望/等待資料：方向可能對，但缺關鍵證據或價格不夠好。

用 3-5 句說明：現在做這個動作，最大的代價是什麼。

### 十、資料來源與可信度

列出實際使用來源：

| 來源 | 日期 | 類型 | 用途 | 可信度 |
|------|------|------|------|--------|
| 財報狗 | | 核心資料源 | 股價/財報/財務指標/估值/法說逐字稿 | A |
| 公司 10-Q/季報 | | 原始資料 | 財務 | A |
| 公司法說 | | 原始資料 | guidance | A |
| vault 報告 | | 歷史研究 | thesis 演變 | B |
| 券商報告 | | 二手分析 | 預期與估值 | B |
| 新聞 | | 二手資料 | 事件 | C |

證據分級：
- A：財報狗可見的股價、財報、財務指標、估值與美股法說逐字稿；公司公告、法說、財報、交易所、監管機構。
- B：券商、定錨、可信研究機構。若引用的是判斷或預估，需與公司資料或財報狗交叉驗證；若只是用來理解市場預期，可明確標註為市場觀點。
- C：新聞、訪談、供應鏈傳聞，需保留不確定性。
- D：同業類比或推論，只能當假設，不可當結論。

### 十一、Vault 覆蓋度與下一步

說明這次讀到哪些 Obsidian 資料：

- 直接相關 ticker/MOC/研究報告數量。
- 最新一份 vault 研究日期。
- 讀到的 Theme/Signal/Daily。
- 觀點衝突與缺口。
- 建議未來補進 vault 的來源。

## 寫作規則

- 先結論，後證據。
- 每個重要數字都有日期與來源。
- 財報狗可見的股價、財報、財務指標、估值與美股法說逐字稿，可直接引用財報狗，不需要額外湊滿 3 個來源；但要優先用已登入 Chrome 受控分頁讀取，先等載入並看畫面，不能用未登入公開頁、第一次文字抽取或 DOM/table count 失敗就判定不可讀。
- 每個買賣判斷都要連到估值或風險報酬。
- 技術名詞先白話再正式。
- 不使用簡體字。
- 不使用方向箭頭符號，以文字說明因果。
- 不使用 Markdown 粗體；標題可以用 `#`。
