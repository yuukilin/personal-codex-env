## 1. 資料源與研究邊界

個股進階研究的目的不是再次介紹公司，而是判斷最新事件是否已經改變公司未來營收、獲利、現金流與估值。

資料優先順序：

1. 財報狗可見的股價、財務、估值與美股法說逐字稿；公司公告、法說、財報、監管申報與交易所資料。
2. 客戶、供應商與競爭對手的原始資料。
3. 具名券商、定錨或專業供應鏈調查。
4. 可靠新聞與訪談。
5. 類別訊號、同業類比與推論。

台美股且財報狗有涵蓋時，採已登入 Chrome 優先。開受控分頁、等待約 5 秒、確認畫面有圖表或「詳細數據」，再讀文字；文字抽不到但畫面可見時用截圖讀表。DOM row/table count 只能輔助，不能單獨判定資料不存在；一種讀法失敗就換文字、畫面或表格區截圖，不反覆重試同一招。預期應有核心資料但因登入或方案權限看不到時停止並回報，不用其他來源假裝完成。若財報狗本來未涵蓋該市場或標的，改讀公司 IR、交易所與監管原始資料。

財報狗個股核心頁優先讀：個股首頁、`/profit-margin`、`/pe`、`/monthly-revenue`、`/monthly-revenue-growth-rate`、`/eps`、`/financial-structure-ratio`。視決策問題補讀：

- `/product-revenue` 可能受方案限制；看不到就明講，不推測產品占比。
- `/long-term-and-short-term-monthly-revenue-yoy`、`/dividend-policy` 可用時作補充。
- `/broker-trading` 只代表券商分點買賣與主力淨買賣超，不等於董監持股、公司基本面或法人共識。

美股法說從 `https://statementdog.com/analysis/{TICKER}/e-report` 找 `/earnings_calls/{id}`，完整讀管理層說法、風險、Q&A 與英文原文；中文摘要與 AI 翻譯只作輔助。財報狗可見的同一筆核心數據可直接引用並標示日期，不必為湊數額外找三個重複來源。

## 3. 正文結構

### 一、先回答：現在最重要的判斷

用完整段落回答，不用大表格：

- 最重要 thesis 是什麼？
- 最新證據使 thesis 變好、變壞、持平或仍不清楚？
- 市場大概已反映多少，依據是什麼？
- 最可能在哪個里程碑與時間窗由財報驗證？
- 現在的行動判斷、信心與最大代價是什麼？

再用 3–5 點列出最重要證據、最大反證與仍缺的資料。

### 二、這次到底改變了什麼

先還原事件時間線與市場原預期：

- 上次研究的核心 thesis 與等待條件。
- 最新法說、財報、公告、客戶或產業資料出現什麼變化。
- 相對公司原 guidance、法人預期與市場敘事，真正超預期或低於預期的是什麼。
- 股價在事件前後相對同業與大盤如何反應。
- 鎖定最新股價的日期、時間與時區。收盤後才發布的財報、公告、法說或客戶消息只能列為「尚未交易的新催化劑」，不得倒過來解釋先前股價。
- 檢查同日公司 IR/SEC 是否有新公告；若沒有，再把產業 ETF、同業共振、利率/大盤、財報前卡位與籌碼列為候選。沒有公司級新資訊時，直接寫清楚，不拿舊消息硬套。

明確排序前三大變化，說明誰是核心驅動、誰只是助燃劑。

### 三、最低必要背景

假設使用者忘記部分細節，用短篇幅交代：

- 公司真正靠什麼產品與客戶賺錢。
- 各主要部門營收與獲利占比。
- 產業週期、訂單到營收的時間差。
- 目前最重要的新舊業務衝突。

不要重寫百科式公司介紹；只補理解本輪 thesis 所需的背景。

### 四、前三大催化劑深挖

每個催化劑各自用完整段落回答 `deep-analysis-core.md` 的催化劑十問。

固定追查：

- 公司是否親自確認？來源原話與上下文是什麼？
- 涉及哪個產品、客戶、地區與部門？
- 商業化目前位於技術能力、送樣、認證、design-in、生態支持/interest、customer forecast/demand、正式訂單/backlog、supply-backed outlook、小量出貨或量產哪一階段？保留原始用詞，不自行升級。
- 下一步還要跨過什麼門檻？通常需要多久？
- 可能貢獻多少營收、毛利與現金流？若不能估，缺什麼資料？
- 舊業務、折舊、費用、價格或競爭會抵銷多少？
- 最強替代解釋與失敗方式是什麼？
- 預估口徑是否一致：單季/全年、segment/全公司、出貨/營收、營收/EPS、實際值/guidance/市場共識不可混用。

前三大催化劑應占主文最大篇幅；不要把它們壓縮成一張表。

### 五、商業化階段與營收橋

先確認每個新成長題材已到哪個階段，不把技術能力當成訂單。

若來源同時給 demand、可供應量與公司 revenue outlook，三者分列；base case 不得高於已取得供應與正式財測支持的部分。超額需求只能列為尚待轉換的 bull-case 選項。

建立公司整體營收橋，至少拆出：

| 項目 | 當前基準 | 下一期變化 | 證據與假設 |
|---|---:|---:|---|
| 既有核心業務 | | | |
| 新產品／新客戶 | | | |
| 價格／匯率／併購 | | | |
| 公司總營收 | | | |
| 毛利率／營益率 | | | |
| EPS／自由現金流 | | | |

表後用段落解釋：新業務對母公司是否重大、何時跨過重大性門檻，以及哪個舊業務可能抵銷。

沒有足夠資料時使用透明區間或敏感度分析，不填假精準數字。

### 六、最新財務品質與管理層可信度

回答：

- 營收、毛利率、營益率、EPS、營業現金流、自由現金流、資本支出與負債的最新變化。
- 成長來自數量、價格、產品組合、併購還是匯率。
- 收入是否含關係人、一次性大單、提前認列或重大合約時點；必要時列出正常化後的成長品質。
- 單一客戶、供應商或同業數據只能當代理指標；先說明對本公司需求的覆蓋範圍與傳導關係，再決定能否升級為公司特定結論。
- ACV、RPO、庫存、應收款、合約負債、backlog 或利用率是否支持管理層說法；指標方向衝突時要解釋，不挑單一好看的數字。
- 新產能的折舊是否先於營收，良率與利用率何時能改善。
- 新業務與舊業務的 gross margin、operating margin、資本強度與營運資金是否不同；若不同就分開建模，不沿用公司歷史 blended margin。
- 高成長率的絕對金額與母公司占比是多少；若公司未揭露基期，只能確認方向，不能直接推 EPS。
- 管理層過去 4–8 季 guidance 的達成度；若經常延後，要降低爆發期信心。
- 商品曝險/週期股另列公司實現價格、銷量、總產能與可用產能、利用率分母、產品組合、單位投入成本、維修/停機與庫存變化；公司實現價與全球/區域基準不同時，解釋產品、地區、合約與認列落差。
- 週期股把最新季度、近十二月與正常化/中景氣 EBITDA、EPS、自由現金流並列，排除一次性收益、保險/和解、合資股東出資與股數下降的放大效果。

每個重要數字標明期間、日期與來源。

### 七、競爭、客戶與供應鏈

不要只列競爭對手名稱。解釋：

- 客戶為什麼選這家公司：技術、成本、認證、交期、可靠度或既有關係？
- 市占提升要從誰手上取得？競爭對手是否更早量產？
- 客戶是否有第二供應商，切換成本與認證週期多長？
- 上游瓶頸、委外產能、良率、原料或設備是否限制放量？
- 客戶集中、單一平台與政策風險如何影響 thesis？

類別供不應求只證明市場環境；必須找到公司料號、客戶、認證、訂單、出貨或 segment 數字，才能升級為公司受惠證據。

### 八、是否已反映與估值

按四層檢查：

1. 股價：事件前後相對大盤與同業的超額報酬。
2. 預期：公司 guidance、法人營收/EPS 共識已算入什麼。
3. 估值：目前倍數相對歷史、同業與獲利品質。
4. 隱含要求：以現在價格反推未來 3–5 年需做到的營收、EPS 或自由現金流，判斷現實性。

依產業選擇 PE、EV/EBITDA、P/S、P/B、FCF yield、SOTP 或 DCF，不硬套同一方法。沒有足夠財測時不要給精確目標價，改寫市場目前要求的營運條件。

商品/週期股不得只套當期低倍數。至少用一組可解釋的中景氣商品價格、利用率、投入成本與維持性資本支出重算正常化獲利，再做反向報酬門檻；若現價需要商品價格長期維持供應衝擊高點，明寫估值已預支高景氣。

公司若提供長期目標，先標示它是 guidance、target/aspiration 還是已簽約收入。再做「即使目標達成」測試：套入至少三個合理終值倍數，計算從現價出發的年化報酬；並反推要達 10% 年化或使用者要求報酬率時，終值仍需多高估值。若報酬仍依賴極高終值倍數，明寫估值已預支成功。

### 九、三情境財務與決策

| 情境 | 真正不確定變數 | 營收／獲利結果 | 合理估值或條件 | 行動 |
|---|---|---|---|---|
| Bear | | | | |
| Base | | | | |
| Bull | | | | |

情境差異要來自真正變數，例如量產時點、滲透率、單價、利用率、毛利率或舊業務衰退，而不是把所有數字一起任意調高或調低。

說明目前最接近哪個情境、從現在切換到另一情境要看到什麼，以及做出目前行動最大的代價。

### 十、替代解釋、反方 thesis 與翻案條件

至少提出：

- 最強替代解釋：同一股價或財務現象還可能由什麼造成。
- 反方 thesis：為什麼公司可能只有題材重評，沒有財務轉折。
- 至少 3 個可觀察翻案條件。

反證要具體，例如「兩季後新產品仍未列入 segment 成長、利用率未升、毛利率反而下降」，不要只寫競爭加劇或景氣轉差。

### 十一、驗證路線與追蹤日曆

把爆發期拆成里程碑：

> 認證 → 訂單 → 小量出貨 → 量產 → 利用率／毛利率改善 → EPS／現金流驗證

這條鏈只作研究底稿；正式報告用完整句子描述各里程碑與時間差。

逐一寫出預計時間、可觀察來源、通過門檻與延誤後如何調整 thesis。追蹤內容要具體到月營收、segment growth、book-to-bill、backlog、利用率、毛利率、客戶認證或法說 guidance。

### 十二、資料來源、Vault 覆蓋與仍不知道什麼

列出實際使用的來源、日期、用途與證據等級。說明：

- 讀到哪些 Ticker MOC、Theme、Signal、Daily、法說與研究報告。
- 最新一份 vault 研究日期與舊 thesis。
- 哪些觀點互相衝突，如何處理。
- 哪些關鍵問題目前公開資料仍無法回答。
- 下一次要補什麼原始來源。

## 4. 個股專用分析規則

- 子公司、品牌、轉投資或事業部的重大性要回到母公司營收、獲利與資本配置；母公司法說視為主來源完整閱讀。
- 新產能不是營收。先查客戶認證、良率、利用率、訂單、折舊與爬坡時程。
- M&A 的策略協同與短期財務稀釋分開；分析融資、整合成本、交叉銷售時點與失敗風險。
- 漲價要分清楚產業報價、公司實際合約、產品組合、轉嫁能力與數量彈性。
- 管理層定性語言不直接進財測；只有可追蹤里程碑才進情境假設。

## 5. 買賣決策規則

- 買入：base case 上行顯著高於下行，關鍵假設已有公司特定證據，估值尚未完全反映。
- 加碼：原 thesis 被新數據強化，重大性提高，估值與風險報酬仍合理。
- 續抱：thesis 未破壞，但新買安全邊際不足，或需要等待下一個財務驗證。
- 減碼：好消息大多已反映、預期過高，或新風險使風險報酬惡化。
- 賣出：核心 thesis 被公司數據推翻，或估值要求遠超合理 bull case。
- 觀望／等待資料：方向可能正確，但缺訂單、量產、重大性、財測或價格安全邊際。

## 6. 品質檢查

除共用完成門檻外，再確認：

- 沒有把技術、產能、送樣或 design-in 寫成已量產營收。
- 新成長已和舊業務衰退放在同一個公司營收橋。
- 公司特定 thesis 有公司、客戶、供應商或具名調查支撐。
- 爆發期有逐步里程碑，且考慮認證、良率、產能與折舊。
- 「尚未反映」有共識、估值或隱含預期證據。
- 情境分析可回推營運假設，不是任意目標價。
- 所有用來解釋股價的事件都早於價格截止點；收盤後消息已另列為尚未交易。
- 已先核對同日公司公告、同業/產業 ETF 與大盤；沒有公司新資訊時，沒有用舊消息假裝成當日催化。
- support、interest、forecast、demand、commitment、order、backlog、supply-backed outlook 與 revenue 沒有混用。
- 已檢查絕對基期、關係人/一次性收入、ACV/RPO/backlog/現金流，以及新舊業務 margin 是否可比。
- 商品曝險/週期股已對齊區域基準、公司實現價、銷量、可用產能、投入成本與正常化獲利，沒有用週期高點低倍數直接判斷便宜。
- 單季/全年、segment/全公司、出貨/營收與實際/預測沒有混口徑。
