# 個股研究報告模板

以下為完整的個股研究報告結構。每個 `##` 區塊都必須包含，除非標注「視情況省略」。

---

## 報告正文結構

```markdown
# [公司名稱]([股票代號]) 深度研究報告

> 報告日期：YYYY-MM-DD
> 資料來源：Obsidian 歷史研究資料庫 + 網路公開資訊
> 報告類型：個股深度研究

---

### 基本資訊

| 項目 | 內容 |
|------|------|
| 公司全名 | xxx |
| 股票代號 | xxx |
| 上市交易所 | xxx |
| 成立年份 | xxx |
| 總部所在地 | xxx |
| 董事長/執行長 | xxx |
| 員工人數 | xxx |
| 最新股價 | xxx（截至 YYYY-MM-DD，來源：網路）|
| 市值 | xxx |

### 公司定位與核心業務

用 3-5 段描述公司做什麼、在產業鏈中扮演什麼角色、核心競爭優勢是什麼。
技術概念要用高中生聽得懂的方式解釋。

引用 vault 中的描述時用 [[wikilink]] 標注來源：
「根據 [[2025-12-05-定錨-xxx]] 的分析，...」

---

## 二、營收結構與產品線

### 營收項目拆解

| 產品/業務線 | 營收占比 | 年增率 | 說明 |
|------------|---------|--------|------|
| xxx | xx% | xx% | 簡述 |
| xxx | xx% | xx% | 簡述 |
| xxx | xx% | xx% | 簡述 |

### 各產品線詳細說明

逐一說明每個營收項目：
- 這個產品/服務是什麼（白話解釋）
- 主要客戶是誰
- 成長驅動力是什麼
- 佔比趨勢（上升/穩定/下降）

資料優先來自 vault 中的個股報告和法說會摘要。

### 客戶與地區分布（若有資料）

主要客戶集中度、營收地區分布。

### 護城河分析

公司的長期競爭優勢是什麼：技術門檻、客戶黏性、規模經濟、專利等。

---

## 四、供應鏈定位

### 產業鏈位置

用文字描述公司在供應鏈中的位置。
引用 vault 報告中的 supply_chain 欄位資料。

上游（原料/設備供應商）：列出關鍵供應商
本公司位置：中游製造/下游應用/等
下游（客戶/終端應用）：列出關鍵客戶和應用場景

### 供應鏈風險與機會

- 是否有供應商集中度風險
- 是否有客戶集中度風險
- 供應鏈中的瓶頸環節
- 潛在的供應鏈變動（如轉單、去中化等）

引用 vault 中 related_tickers 的報告，找出上下游公司的動態。

## 六、成長動能與催化劑

### 短期催化劑（未來 3 個月）

從 vault 報告的 catalysts 欄位和內文中提取。
用網路資料補充最新的事件日曆。

### 中長期成長動能

引用 vault 中的 Theme 卡片和產業報告，說明支撐公司成長的結構性趨勢。

### vault 中的觀點演變

如果不同時期的報告對成長展望有不同看法，呈現演變：
「2025-06 時，定錨認為...；到了 2025-12，觀點修正為...；最新的 2026-03 報告則指出...」

---

## 七、風險因素

### 基本面風險

- 產業景氣下行
- 客戶訂單流失
- 技術落後
- 成本上升

### 外部風險

- 地緣政治（中美關係、出口管制等）
- 政策法規變動
- 匯率波動
- 原物料價格

### vault 中標記的衝突觀點

若 vault 中有 conflict_flag = true 的報告，在此呈現不同方的觀點。

---

## 八、投資智慧框架套用

從使用者 vault 的 `1 Cards/` 投資智慧卡片中，挑選與本標的目前狀態最相關的 2-4 張卡片，說明為何適用。

### 股價生命週期判斷

根據目前的營收趨勢、股價位階、本益比變化，判斷這支股票處於哪個階段。
引用 [[股價生命循環 - 買在上升階段]] 或其他相關卡片的框架。

### 適用的交易哲學

列出 2-3 張與當前情況最吻合的投資智慧卡片，每張說明：
- 卡片名稱（用 [[wikilink]]）
- 為什麼現在適用這張卡片
- 對應到這支股票的具體情境是什麼

範例：
如果定錨連續三季上修 EPS，引用 [[獲利擊敗預期，會延續跟擴散]]，說明這代表基本面超預期的訊號可能持續。
如果股票剛從盤整區突破，引用 [[最好的上漲，要先經過一段洗盤]] 和 [[突破後可能立刻拉回，要給股票一些活動空間]]。

注意：不做買賣建議，只呈現框架與現狀的對應關係，供使用者自行判斷。

## 九、跨報告發現（本報告最大附加價值）

這一節是整份報告的精華。整合 vault 搜尋過程中發現的：

### 多報告交叉驗證的高確信度結論

被 3 份以上報告反覆提及的論點或趨勢。列出每個結論的佐證報告（用 [[wikilink]]）。

### 跨主題的意外連結

此標的同時出現在哪些不同的 Theme 中，這些主題的交叉代表什麼意義。
例如：家登同時出現在 advanced-packaging 和 semiconductor-cycle 兩個主題，代表它既受惠於結構性趨勢，也受週期影響。

### 觀點衝突彙整

如果 vault 中不同報告對本標的有衝突觀點（特別是 conflict_flag = true 的報告），在此完整呈現雙方論點，不做裁判。

### 資訊缺口提醒

vault 中缺少哪些面向的研究，建議使用者未來補充。

## 十一、結論與觀察

用 5-8 句話總結：
1. 這家公司值不值得深入追蹤
2. 目前處於什麼階段（成長初期/加速/成熟/轉型）
3. 關鍵的觀察指標是什麼
4. vault 資料庫中最需要更新的是什麼

注意：不做買賣建議，只提供研究觀點。
```

## 寫作注意事項

### 引用 vault 資料的格式

- 引用特定報告：「根據 [[2025-12-05-定錨-xxx]] 的分析」
- 引用 Theme 卡片：「[[Theme-advanced-packaging]] 卡片記錄」
- 引用 Ticker 總整理：「[[2330.TW-研究總整理]] 頁面顯示」
- 多份報告佐證：「共有 N 份報告（[[報告1]]、[[報告2]]、[[報告3]]）提到此趨勢」

### 引用網路資料的格式

- 「根據網路公開資訊（來源：MoneyDJ，2026-04-20），最新股價為...」
- 「Seeking Alpha 2026-04-18 報導指出...」

### 時效性標注

- vault 資料：「根據你 2025-11 月的定錨報告」
- 過時提醒：「這份報告已超過 6 個月，數據可能已過時」
- 網路資料：「截至 2026-04-22 的最新數據」
