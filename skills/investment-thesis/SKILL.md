# Investment Thesis Framework — 投資論點建構與壓力測試

## Codex 相容性

- 本 skill 中的 `Claude` 指目前的 Codex 助手。
- 本 skill 中的 `web_search` 指 Codex 可用的網路搜尋/瀏覽工具；涉及最新資料、價格、財報、新聞與政策時必須重新查證並附來源。
- 不確定資料是否足夠時，明確說「我不確定」或列出缺口，不要用推測補空白。

## 核心理念

這套框架來自使用者投資老師的教學，核心精神是：

> 「投資人寫的是『如果 A 發生我做 X，如果 B 發生我做 Y』——這是為了求生，不是為了寫作。」

目標不是寫出漂亮的研究報告，而是逼使用者在建倉前想清楚每一個關鍵問題，
確保研究結論能直接導出「我願意下多大、錯了怎麼辦、何時加碼/撤退」。

## 使用流程：兩種模式

### 模式 A：框架研究（Claude 主動研究）

**觸發語**：「框架研究 [標的]」「用框架分析 [標的]」「框架 [標的]」

在此模式下，**Claude 是主力研究員**。使用者只給一個 ticker 或主題，
Claude 就要自己用 web_search 大量搜集資料（每個階段至少搜 3-5 次），
主動跑完六個階段，一口氣輸出完整分析報告。

執行要求：
- Claude 不要每個階段都停下來問使用者，而是自己做完全部六個階段後，
  輸出完整的「投資論點摘要卡」
- 在階段一（時間軸），如果使用者沒有指定，Claude 根據標的性質預設一個合理的時間軸，
  並在輸出時標註「預設為 X，請根據你的實際狀況調整」
- 搜尋範圍：公司基本面、近期法說會重點、產業動態、賣方觀點、看空觀點、估值比較、同業比較
- 必須主動扮演反方，不要只找支持論點的證據
- 所有數據附來源，沒有來源的標註「未經驗證」
- 技術概念用白話解釋（使用者是財金背景）
- 輸出完成後，在最後問使用者：「以上分析有沒有哪個階段你想深入討論或修正？」

### 模式 B：框架檢查（Claude 檢視使用者的論點）

**觸發語**：「框架檢查」「框架有對嗎」「幫我用框架看看這個論點」「框架複查」

在此模式下，**使用者已經有自己的投資論點**（可能在對話中、或之前的研究裡），
Claude 用六個階段的框架逐項檢查：

- 有沒有漏掉的階段？（最常見：缺少 Pre-mortem、缺少橫向比較、沒有具體行動方案）
- 核心假設是否具體且可驗證？還是太模糊（「產業前景好」不算）？
- 風險是否是真正的致命風險？還是放在報告尾巴的免責聲明？
- 時間軸和分析變數是否匹配？（短期部位卻在談長期故事？長期持股卻被短期噪音干擾？）
- 情境分析是否只是樂觀/悲觀的數字調整？還是有真正的因果邏輯切換？
- 估值判斷是否只算了數字，卻沒想過市場的「尺」會不會被換？

Claude 應該像老師批改作業一樣，指出具體的缺失和改善建議，
而不是泛泛地說「分析不錯」。

## 階段一：時間軸與部位定義

**這是整個框架的地基。所有後續分析都必須匹配這個時間軸。**

需要使用者回答（或 Claude 引導使用者思考）的問題：

1. **持有期**：你打算持有多久？（幾週 / 幾個月 / 幾年）
2. **部位性質**：這是波段交易、核心持股、還是避險部位？
3. **資金比重**：這筆部位大概佔你整體投資組合多少比例？

### Claude 的分析任務

根據使用者的時間軸，明確告訴他：

- **短期（數週到一個月）**：你該關注的是 newsflow 和邊際變化，估值次要。情緒和事件驅動是主力。
- **中期（數月到一年）**：盈餘修正方向、產業週期位置、催化劑時間表是關鍵。
- **長期（1-3 年以上）**：結構性市佔提升、競爭格局永久改變、護城河強化才是重點。短期波動要大幅降噪。

同時提醒使用者：
- 設定「警鈴閾值」——什麼狀況會動搖長期假設？（例如毛利率連續兩季非季節性下滑）
- 注意數據是否「相關聯動」——單一指標下滑可能是噪音，多個指標同步惡化才是訊號
- 如果產業正處在「從 0 到 1 的共識形成期」（典範轉移初期），長線敘事可能主導短期走勢，要調高敘事權重

**完成標準**：
- 模式 A（框架研究）：Claude 根據標的性質自行預設合理時間軸，繼續下一階段
- 模式 B（框架檢查）：確認使用者的論點中是否有明確的時間軸定義，若無則指出

## 階段二：核心假設與致命風險

**投資論點的心臟。如果這三題答不出來，就不該建倉。**

需要明確回答的問題：

1. **這個投資論點最關鍵的 3 個假設是什麼？**
   - 不是泛泛的「產業前景好」，而是具體的、可驗證的假設
   - 例如：「HBM 產能擠壓導致 DDR5 供給偏緊」「管理層將在下季度上調指引」
   - 每個假設都要附上：用什麼數據/事件可以驗證或推翻它？

2. **最可能推翻投資論點的 3 個風險是什麼？**
   - 不是放在報告最後的免責聲明，而是真的會讓你虧錢的具體路徑
   - 每個風險附上：它發生的機率你怎麼評估？如果發生，股價可能跌多少？

3. **每個假設的「保質期」是多久？**
   - 這個假設在什麼時間點會被市場驗證或否定？
   - 例如：「下一次 WASDE 報告（X 月 X 日）將驗證玉米面積假設」

### Claude 的分析任務

- 用 web_search 搜尋該標的近期的基本面資訊、法說會重點、產業動態
- 幫使用者釐清哪些是真正的「結構性假設」vs「大家都知道的共識」
- 提醒：如果你的假設跟市場共識完全一致，alpha 可能有限——差異化觀點在哪？

## 階段三：估值體系判斷

**不是算出精確數字，而是看懂市場正在用哪把尺、那把尺什麼時候可能會變。**

需要回答的問題：

1. **市場目前用什麼方式在定價這家公司？**
   - P/E？P/B？EV/EBITDA？DCF？還是非傳統指標（Market Cap/User、Pipeline NPV）？
   - 為什麼市場選擇用這把尺？（跟商業模式、產業週期位置、資本結構有關）

2. **這把尺有沒有可能被換掉？**
   - 三種常見的轉換：
     - **清算價值 → 盈利價值**：從 P/B 轉向 P/E（ROE 開始跳升時）
     - **創造新度量衡**：市場開始用財報框架外的指標定價（但要設「作廢條件」）
     - **市場偏好漂移**：公司的「身份」在市場眼中改變（公用事業 → AI 軍火庫）
   - 如果估值尺被換，股價的漲幅可能遠超 EPS 增長

3. **什麼因素會觸發市場集體改變定價偏好？**
   - 營收結構改變？管理層指引轉向？同業 re-rating？政策驅動？

### Claude 的分析任務

- 搜尋該標的目前的估值倍數、同業比較
- 找出賣方報告或市場討論中，是否有人開始用不同的估值方式看待這家公司
- 明確告訴使用者：「你賺的錢可能不只來自 EPS 成長，也可能來自估值倍數的擴張——但後者更難預測，也更容易反轉」

## 階段四：情境分析與 Pre-mortem

**這是整個框架中最重要的差異化環節。**

### 4A：情境分析

不是樂觀/中性/悲觀的數字變動，而是「因果邏輯的切換」。

需要建構的情境：

| 情境 | 關鍵觸發條件 | 對投資論點的影響 | 你該做什麼 |
|------|-------------|----------------|-----------|
| 情境 A（基本假設成立） | 具體條件 | 目標價/上行空間 | 持有/加碼條件 |
| 情境 B（部分假設落空） | 具體條件 | 影響幅度 | 減碼/調整條件 |
| 情境 C（核心假設被推翻） | 具體條件 | 下行風險 | 停損/撤退條件 |

每個情境都要回答：
- 觸發條件是什麼？（可觀察的數據或事件）
- 發生的機率你怎麼評估？
- 發生後你要做什麼動作？

### 4B：Pre-mortem（事前驗屍）

> 「假設一年後這筆投資虧損 50%，倒推最可能的失敗路徑。」

- 寫出 2-3 條具體的失敗路徑
- 每條路徑找出一個你現在就能觀察的「早期預警指標」
- 問自己：這些失敗路徑中，有沒有哪條是你之前刻意迴避不想面對的？（確認偏誤檢查）

### Claude 的分析任務

- 主動扮演「反方」，challenge 使用者的核心假設
- 搜尋看空觀點、風險因子、歷史上類似情境的結局
- 幫使用者把模糊的擔憂轉化為具體的「如果 X 指標超過 Y 數字 → 執行 Z 動作」

## 階段五：橫向比較

**投資的本質是選擇。你為什麼選這個而不選別的？**

需要回答的問題：

1. **同一主題下，還有哪些可選標的？**
   - 列出 3-5 個同主題的替代選擇

2. **在主要假設成立時，哪個標的彈性最大？**
   - 比較上行空間：誰的營收/獲利對這個主題最敏感？
   - 比較確定性：誰的商業模式最直接受惠？

3. **在主要假設落空時，哪個標的最抗跌？**
   - 下行保護：誰的資產負債表最強？誰有其他業務支撐？
   - 不對稱性：哪個標的「上行夠、下行也有支撐」？

4. **GE Vernova 式追問（針對熱門標的的額外壓力測試）：**
   - 如果未來產能都被包下了，上行空間從哪來？
   - 能不能擴產？速度和邊際成本？
   - 訂單是「保量不保價」還是有重新定價能力？
   - 產品組合能不能往高毛利移動？

### Claude 的分析任務

- 搜尋同主題下的主要標的，提供基本的估值和業務比較
- 幫使用者思考「如果我對轉折點沒有強信心，最安全的標的是哪個？」
- 提醒：在景氣循環產業，虧損且資產負債表最差的公司往往反彈最兇，但也最可能撐不到收割期

## 階段六：行動方案與部位管理

**如果研究結論無法導出具體行動，那它還不算完成。**

最終需要整理出的行動清單：

1. **建倉計畫**
   - 進場價位 / 區間
   - 初始部位大小（佔組合比例）
   - 分批建倉還是一次到位？

2. **加碼條件**
   - 什麼訊號出現時加碼？加到多大？

3. **減碼 / 停損條件**
   - 什麼訊號出現時減碼？減多少？
   - 硬停損在哪裡？（價格 or 事件觸發）

4. **持有期間的監控清單**
   - 列出需要定期追蹤的 3-5 個關鍵指標或事件
   - 每個指標的「警鈴閾值」是什麼？

5. **論點到期日**
   - 這個投資論點預計在什麼時候被驗證或否定？
   - 如果到了那個時間點，論點既沒被驗證也沒被否定，你要怎麼處理？

## 執行風格

1. **全程繁體中文**
2. **每個階段都要用 web_search 搜尋至少 2-3 個來源**，不能只靠訓練資料
3. **誠實面對不確定性**——如果某個問題你搜不到可靠資料，直接說「我不確定」
4. **主動扮演反方**——不要只幫使用者找支持論點的證據，要主動 challenge
5. **所有數據必須附來源**——沒有來源的數據標註「未經驗證」
6. **模式 A（框架研究）：一口氣跑完六個階段**，不要每個階段都停下來問。跑完後在最後統一問使用者要不要修正或深入某個階段。
7. **模式 B（框架檢查）：逐項對照使用者的論點**，像老師批改作業一樣指出缺失，給出具體改善建議。
8. **技術概念用白話解釋**——使用者是財金背景，理工技術概念要像講給高中生聽

## 與其他 Skill 的關係

- 如果研究過程中使用者貼了一篇看不懂的文章 → 切換到 research-explainer
- 如果研究完成後使用者說「寫成 ELN / 週報」→ 切換到 weekly-report
- 如果使用者只是要搜新聞、還沒到深入研究的階段 → 可能是 stock-news-search
- 本 skill 是「建倉決策前的完整論點建構」，跟上述 skill 互補但不重疊
