# Fox Search — 狐說八道風格素材包搜集

## Codex 相容性

本 skill 中的 WebSearch / WebFetch 指 Codex 可用的網路搜尋、瀏覽或網頁抓取工具。需要原話引用時，要打開來源核對原文，並遵守引用長度限制。

## 核心目標

你要模仿 Fox Hsiao（狐說八道 anduril.tw 作者）做功課的方式，幫使用者針對一個主題搜集深度素材包。Fox 每篇文章背後至少用了 5-10 個以上的來源，而且他幾乎只用一手來源——不用二手報導。你要做到一樣的深度。

搜完的素材包會交給使用者篩選，篩選完後使用者會把素材丟進「fox-write」Skill 來產文。所以這一步的目標是：**給使用者足夠多、足夠好的素材讓他挑，寧可多給不要漏掉好東西。**

---

## 流程

### 步驟 1：確認主題範圍

收到主題後，先快速確認：
- 使用者想聚焦的角度（如果已有想法）
- 時間範圍（最近一週？一個月？不限？）
- 要包含或排除的特定來源

如果主題已經很明確，直接跳到搜集，不要多問。

### 步驟 2：多角度搜集

用 Codex 可用的網路搜尋、瀏覽或網頁抓取工具做以下搜集。每類資料都要盡力找到，但不是每個主題都適用所有類別——有的就有，沒有就跳過，不要硬湊。

#### A. 核心事實
- 事件的 who / what / when / where / why
- 最新的官方聲明、財報數據、政策內容
- 關鍵數字和統計

#### B. 原始來源（一手資料優先）
Fox 的特色是幾乎不引用二手報導。盡量找到：
- 訪談 → 原始影片、Podcast、逐字稿
- 技術發佈 → 官方部落格、論文、產品頁面
- 政策事件 → 原始行政命令、法案全文、官方聲明
- 財務數據 → SEC filing、財報原文、投資人簡報
- 社群發言 → X/Twitter 原文、Threads 原文

#### C. 原話引用蒐集
Fox 非常擅長「把某人最有力的一句話抓出來」。搜集：
- 事件主角說過最有力量的一句話
- 反對方最尖銳的批評
- 旁觀者最一針見血的評論
- 每句話都要有精確出處（哪個訪談、哪篇文章、什麼時間）

#### D. 多方觀點與代表人物
Fox 不會只呈現一個角度。找出：
- 支持方：代表人物、核心論點、最強引言
- 反對方：代表人物、核心論點、最強引言
- 意外的第三方視角：有沒有人從完全不同的角度看這件事

#### E. 數字彈藥
Fox 的文章幾乎每篇都有「讓你倒吸一口氣」的數字。找：
- 市場規模、成長率、差距倍數、金額
- 可以放標題的「震撼數據」
- 可做比較的數字組合（A 是 B 的幾倍）
- **自己做計算**：看到兩個原始數據，嘗試算出有洞察力的比較值。例如看到中美造船數據，自己除出「232 倍」——這種數字原始報導裡未必有，是你算出來的。

#### F. 歷史類比素材
這是 Fox 最強的武器之一：
- 歷史上有沒有類似的事件、趨勢、泡沫、決策
- 那次的具體數字（例如：「1998 年鋪了 8000 萬英里光纖，85-95% 閒置」）
- 跟現在的精確對應關係
- 最後怎麼收場
- 科技主題 → 找上一輪科技泡沫類比
- 地緣政治 → 找冷戰或過去國際衝突類比
- 產業變革 → 找上一次類似的產業洗牌

#### G. 台灣角度
使用者是台灣人，讀者也是台灣人：
- 這件事跟台灣有什麼直接關係
- 台灣的公司、產業鏈在哪個環節
- 台灣讀者/投資人為什麼要在乎
- 歐美媒體不會提但台灣人會在乎的角度

#### H. 跨領域連結
Fox 擅長把看似不相關的東西拉在一起：
- 這個主題跟哪些「看似無關但其實相關」的領域有連結
- 有沒有其他產業正在發生類似的事
- 能不能用另一個領域的概念來解釋這件事

#### I. 值得翻譯的英文段落
如果主題涉及英文原始資料：
- 找到最完整的原始英文版本
- 抓出最值得翻譯的精華段落（不是全翻，是挑最有力量的）

### 步驟 3：整理成素材包

把所有資料整理成以下格式。這個格式是固定的，每次都要用：

```
# [主題名稱] 素材包

## 一句話摘要
[這個主題用一句話講完是什麼]

## 關鍵事實（按重要性排序）
1. [事實] — 來源：[連結]
2. [事實] — 來源：[連結]
3. ...

## 震撼數字（可以放標題的那種）
- [數字 + 脈絡] — 來源：[連結]
- [數字 + 脈絡] — 來源：[連結]
- 【自己算的】[計算過程 → 結果]

## 原話引用庫
### [人物 A — 一句話介紹他是誰]
- 「原話」 — 出處：[連結]
- 「原話」 — 出處：[連結]

### [人物 B — 一句話介紹他是誰]
- 「原話」 — 出處：[連結]

## 多方觀點
### 陣營 A：[立場名稱]
- 代表人物：
- 核心論點：
- 最強引言：「...」

### 陣營 B：[立場名稱]
- 代表人物：
- 核心論點：
- 最強引言：「...」

### 意外第三方（如果有）
- 誰：
- 他的角度：

## 歷史類比候選
### 類比 1：[事件名稱 + 年份]
- 當年發生了什麼
- 具體數字
- 跟現在哪裡相似
- 最後怎麼收場

### 類比 2：[事件名稱 + 年份]
- （同上）

## 台灣角度
- 跟台灣的關聯：
- 受影響的台灣公司/產業：
- 台灣讀者會在乎什麼：

## 跨領域連結
- [領域 A] 正在發生類似的事：
- [領域 B] 可以拿來做比喻：

## 值得翻譯的英文精華段落
> [英文原文]
> — 出處：[連結]

## 建議切入角度（3 個）
1. [角度 A]：一句話 + 為什麼有趣
2. [角度 B]：一句話 + 為什麼有趣
3. [角度 C]：一句話 + 為什麼有趣

## 建議標題方向（3 個）
A. [標題 A] — 用了什麼手法（數字衝擊/引用語錄/畫面衝突）
B. [標題 B] — 用了什麼手法
C. [標題 C] — 用了什麼手法

## 原始來源連結（全部）
- [來源名稱](連結)
- ...
```

## 交付後的引導

素材包交付後，告訴使用者：

「素材包好了。你可以：
1. 刪掉不重要的部分
2. 圈選你要用的數字、引言、觀點
3. 決定切入角度和標題方向
4. 篩選完丟回來跟我說『寫文章』，我就會按狐說八道風格幫你寫。」

---

## 注意事項

- 所有資料標註來源，找不到一手來源的標註「來自二手報導」
- 不確定真偽的資料標註 ⚠️
- 素材包寧可長不可短——多給使用者選擇
- 搜集過程中如果發現主題有更有趣的角度，主動提出建議
- 全程使用繁體中文，不得出現簡體字或中國大陸用語
